# Sample Job Descriptions for Testing

Use these sample job descriptions to test the Tech Stack Analyzer:

---

## Sample 1: Full Stack Developer

```
Senior Full Stack Developer - Remote

We are seeking an experienced Full Stack Developer to join our fast-growing startup. You will be responsible for building scalable web applications from the ground up.

Requirements:
- 5+ years of experience with React and TypeScript
- Strong proficiency in Node.js and Express
- Experience with PostgreSQL or MongoDB
- Familiarity with AWS services (EC2, S3, Lambda)
- Knowledge of Docker and container orchestration
- Experience with CI/CD pipelines (GitHub Actions, Jenkins)
- Understanding of RESTful APIs and GraphQL
- Version control with Git

Nice to Have:
- Experience with Next.js or other React frameworks
- Knowledge of Redis for caching
- Familiarity with microservices architecture
- Experience with Kubernetes
- Testing frameworks (Jest, React Testing Library)

This role offers competitive salary, remote work, and equity options.
```

## Sample 3: DevOps Engineer

```
DevOps Engineer - Cloud Infrastructure

We're looking for a DevOps engineer to help us scale our infrastructure and improve our deployment processes.

Must Have:
- 3+ years experience with AWS (EC2, ECS, EKS, RDS, S3)
- Strong knowledge of Docker and Kubernetes
- Infrastructure as Code (Terraform or CloudFormation)
- CI/CD pipeline setup (GitHub Actions, GitLab CI, or Jenkins)
- Linux system administration
- Bash/Python scripting
- Monitoring and logging (Prometheus, Grafana, ELK stack)

Good to Have:
- Experience with Azure or GCP
- Configuration management (Ansible, Chef, Puppet)
- Service mesh (Istio, Linkerd)
- Security best practices and compliance
- Database management and optimization
- Experience with serverless architecture

We offer a collaborative environment and opportunities for continuous learning.
```

## How to Use

1. Copy any of the above job descriptions
2. Paste into the Tech Stack Analyzer input field
3. Click "Analyze Tech Stack" or press Ctrl+Enter
4. Review the generated tech stack breakdown and learning roadmap

## Tips for Testing

- Try different types of roles (frontend, backend, full-stack, etc.)
- Mix explicit and implicit technology requirements
- Test with varying levels of detail
- Try international job postings with different terminology
- Experiment with combining multiple technologies
